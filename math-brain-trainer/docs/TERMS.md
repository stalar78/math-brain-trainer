# TERMS

## Назначение
Этот файл фиксирует основные термины проекта Math Brain Trainer.

Он нужен для того, чтобы:
- не путаться в именах сущностей;
- одинаково называть вещи в коде и документации;
- не создавать дублирующие понятия;
- упростить архитектуру и коммуникацию с GPT.

---

## Основные продуктовые термины

### Math Brain Trainer
Название проекта.

Это веб-приложение для тренировки математического мышления, устного счёта и вычислительных техник.

---

### Training
Одна активность пользователя внутри тренажёра.

Обычно training — это процесс прохождения серии задач по выбранному режиму.

---

### Mode
Режим тренировки.

Mode отвечает на вопрос:
**какой тип математической практики сейчас проходит пользователь?**

Примеры:
- addition
- subtraction
- multiplication
- division
- mixed

Важно:
mode — это верхний уровень организации тренировки.

---

### Technique
Конкретный вычислительный приём внутри режима.

Technique отвечает на вопрос:
**какой именно способ мышления или вычисления тренируется?**

Примеры:
- make tens
- round numbers
- complement method
- multiply by 5
- multiply by 11
- divide by 5 rule

Важно:
technique не равна mode.

Правильно:
- mode = addition
- technique = make tens

Неправильно:
- mode = make tens

---

### Task
Одна математическая задача, которую видит пользователь.

Task обычно содержит:
- операнды;
- оператор;
- текст или выражение;
- правильный ответ;
- метаданные сложности;
- привязку к mode и, при необходимости, technique.

Пример:
- 27 + 8
- 64 - 29
- 25 × 11
- 120 ÷ 5

---

### Answer
Ответ пользователя на конкретную задачу.

Answer может быть:
- числовым;
- введённым в input;
- проверяемым на корректность.

---

### Correct Answer
Правильный ответ задачи, рассчитанный системой.

---

### Session
Одна завершённая тренировочная сессия.

Session — это набор задач, решённых подряд в рамках одного запуска тренировки.

Session обычно включает:
- выбранный режим;
- параметры сложности;
- список задач;
- ответы пользователя;
- время;
- количество правильных ответов;
- итоговый результат.

---

### Session Result
Итог одной тренировочной сессии.

Session Result отвечает на вопрос:
**как пользователь прошёл эту конкретную тренировку?**

Обычно включает:
- total tasks
- correct answers
- wrong answers
- accuracy
- duration
- score
- mode
- technique
- date

---

### Stats
Накопленная статистика пользователя.

Stats отвечает на вопрос:
**что в целом известно о прогрессе пользователя за всё время?**

Обычно включает:
- total sessions
- total tasks solved
- average accuracy
- best score
- streaks
- per-mode stats

---

### Progress
Общий рост результатов пользователя во времени.

Progress может отражаться через:
- рост точности;
- рост скорости;
- увеличение сложности;
- улучшение по конкретным техникам;
- серию регулярных тренировок.

---

## Архитектурные термины

### Entity
Доменная сущность проекта.

Примеры:
- TrainingMode
- TechniqueType
- MathTask
- TrainingSession
- SessionResult
- UserStats

Entity — это не просто кусок данных, а важное понятие предметной области.

---

### Feature
Функциональный блок приложения, реализующий пользовательский сценарий.

Примеры:
- mode selection
- task generation
- answer checking
- results calculation
- stats persistence

Feature — это уровень бизнес-логики и поведения.

---

### Component
Переиспользуемый элемент интерфейса.

Примеры:
- button
- mode card
- answer input
- task display
- result summary

Component — это в первую очередь UI.

---

### Page
Целый экран приложения.

Примеры:
- HomePage
- ModeSelectPage
- TrainingPage
- ResultPage
- StatsPage

Page собирает несколько компонентов и фич в единый экран.

---

### Service
Слой для работы с внешними механизмами хранения или инфраструктурой.

На этапе MVP это в первую очередь:
- localStorage services

Примеры:
- saveStats
- loadStats
- saveSessionHistory

---

### Utility / Utils
Небольшие вспомогательные функции общего назначения.

Примеры:
- random number helpers
- number formatting
- clamp
- shuffle

Важно:
utils не должны превращаться в свалку бизнес-логики.

---

## Математические термины проекта

### Addition
Режим тренировки сложения.

На старте может включать:
- базовое сложение
- поиск десяток
- круглые числа
- разбиение на части
- группировку одинаковых чисел

---

### Subtraction
Режим тренировки вычитания.

На старте может включать:
- базовое вычитание
- дополнение
- разбиение на части
- вычитание из 100 / 1000 / 10000

---

### Multiplication
Режим тренировки умножения.

На старте может включать:
- базовое умножение
- умножение на 5
- умножение на 11

---

### Division
Режим тренировки деления.

На старте может включать:
- базовое деление
- деление на 2 через группы
- деление на 5

---

### Mixed Mode
Режим, который смешивает несколько типов операций.

Этот режим не обязателен для MVP, но может быть добавлен позже.

---

### Difficulty
Уровень сложности задач.

Difficulty может определяться через:
- диапазон чисел;
- количество шагов;
- тип техники;
- скорость сессии;
- наличие отвлекающих вариантов;
- длину вычисления.

Примеры значений:
- easy
- medium
- hard

---

### Speed Mode
Режим, где акцент делается на скорость решения.

---

### Technique Mode
Режим, в котором тренируется конкретный вычислительный приём.

Пример:
не просто addition, а addition + make tens.

---

## Термины хранения и данных

### localStorage
Стартовый механизм хранения пользовательского прогресса в браузере.

Используется для:
- статистики;
- рекордов;
- настроек;
- истории сессий.

---

### Persistence
Общий термин для сохранения данных между запусками приложения.

На MVP persistence = localStorage.

Позже persistence может быть расширен до облачного хранения.

---

### Session History
История завершённых тренировочных сессий.

Используется для:
- просмотра недавних результатов;
- анализа прогресса;
- построения статистики.

---

## Термины UX

### Feedback
Обратная связь интерфейса на действие пользователя.

Примеры:
- правильный ответ;
- неправильный ответ;
- завершение сессии;
- новый рекорд;
- ошибка ввода.

---

### Flow
Ощущение непрерывности прохождения тренировки.

Хороший flow означает:
- минимум лишних кликов;
- понятный переход между задачами;
- быстрый ввод ответа;
- ясный результат.

---

### Session Length
Размер одной тренировочной сессии.

Например:
- 10 задач
- 20 задач
- 60 секунд
- 3 минуты

---

## Правила использования терминов

### 1. Не смешивать mode и technique
Правильно:
- addition = mode
- make tens = technique

### 2. Не смешивать task и session
Правильно:
- task = одна задача
- session = набор задач

### 3. Не смешивать result и stats
Правильно:
- result = итог одной сессии
- stats = накопленные итоги пользователя

### 4. Не использовать несколько слов для одной сущности
Если выбрано слово `task`, не нужно потом параллельно использовать:
- example
- problem
- exercise
в качестве основной технической сущности.

Допустимо использовать их в текстах интерфейса, но не как конкурирующие доменные имена в коде.
