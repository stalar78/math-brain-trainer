# START_HERE

## Зачем нужен этот файл
Этот файл — краткая точка входа в проект Math Brain Trainer.

Он нужен для того, чтобы после паузы быстро понять:
- что это за проект;
- где находится основная документация;
- в каком порядке начинать работу;
- какой следующий практический шаг.

---

## Что это за проект
Math Brain Trainer — это веб-приложение на React + TypeScript + Vite для тренировки мозга через математические задачи.

Главная идея проекта:
не просто давать случайные примеры, а развивать вычислительные навыки и математические техники.

Базовые направления проекта:
- addition
- subtraction
- multiplication
- division

---

## Основной принцип разработки
Проект разрабатывается пошагово с помощью GPT.

Формат работы:
1. GPT предлагает один следующий логичный шаг.
2. GPT объясняет, что мы делаем.
3. GPT показывает, какие файлы создать или изменить.
4. GPT даёт полный код файлов.
5. Пользователь вручную создаёт файлы в VS Code и вставляет код.
6. Пользователь запускает проект и проверяет результат.
7. После этого начинается следующий шаг.

## Что делать в первый рабочий день
Практический старт проекта должен быть таким:

### Шаг 1. Создать проект
Создать новый проект:
- React
- TypeScript
- Vite

### Шаг 2. Подключить стили
Установить и настроить:
- Tailwind CSS

### Шаг 3. Очистить шаблон
Убрать стандартный Vite-шаблон и подготовить чистую основу.

### Шаг 4. Создать структуру папок
Подготовить внутри `src/` базовую структуру:
- app/
- pages/
- components/
- features/
- entities/
- shared/
- services/
- utils/
- data/

### Шаг 5. Сделать стартовый UI-каркас
Подготовить:
- базовый layout
- HomePage
- экран выбора режима

## Что НЕ делать на старте
Пока не нужно:
- backend
- авторизация
- облачная синхронизация
- сложный state manager
- слишком сложная маршрутизация
- adaptive difficulty
- achievements
- analytics

Сначала нужен чистый рабочий MVP.
