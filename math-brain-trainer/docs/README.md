# Math Brain Trainer

## О проекте
Math Brain Trainer — это веб-приложение для тренировки мозга через решение математических задач и развитие навыков устного счёта.

Проект создаётся как личный математический тренажёр, но с возможностью дальнейшей публикации онлайн.

## Главная идея
Это не просто генератор случайных примеров, а тренажёр математических техник и вычислительных навыков.

Приложение должно помогать:
- ускорять устный счёт
- улучшать точность вычислений
- тренировать устойчивые вычислительные паттерны
- развивать уверенность при работе с числами
- отслеживать прогресс тренировок

## Базовые направления
Первая версия проекта включает 4 фундаментальных направления:

- Addition
- Subtraction
- Multiplication
- Division

## Техники, которые планируется развивать

### Addition
- поиск десяток
- поиск круглых чисел
- группировка одинаковых чисел
- разбиение на части

### Subtraction
- вычитание через дополнение
- разбиение на части
- быстрое вычитание из 100 / 1000 / 10000

### Multiplication
- умножение на 5
- умножение на 11

### Division
- деление на 2 через группы
- деление на 5 через правило "сначала на 10, потом умножить на 2"

## Почему выбран именно этот стек
- React удобен для интерактивного интерфейса
- TypeScript помогает держать проект в порядке
- Vite даёт быстрый старт и удобную разработку
- Tailwind ускоряет создание аккуратного интерфейса
- localStorage позволяет обойтись без backend на первом этапе

## Подход к разработке
Проект разрабатывается пошагово с помощью GPT в следующем формате:
1. GPT предлагает один логичный шаг.
2. GPT объясняет, что именно делается.
3. GPT показывает, какие файлы нужно создать или изменить.
4. GPT выдаёт полный код файлов.
5. Пользователь вручную создаёт файлы в VS Code и вставляет код.
6. Пользователь запускает проект и проверяет результат.
7. После этого переходим к следующему шагу.

## Важные правила разработки
- не перепрыгивать через этапы
- сначала делать рабочий MVP
- не усложнять архитектуру раньше времени
- не добавлять backend на старте
- не внедрять тяжёлые state-менеджеры без необходимости
- хранить генерацию задач отдельно от UI
- все изменения делать малыми понятными шагами
